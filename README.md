#### git commit 命名规范
| 类型 | 备注 |
|:-------- |:---- |
| feat     | 新功能（feature）|
| fix      | 修补bug|
| docs     | 文档（documentation）|
| style    |  格式（不影响代码运行的变动）|
| refactor | 重构（既不是新增功能，也不是修改bug的代码变动）|
| test     | 增加测试代码 |
| chore    | 构建过程辅助工具的变动 |
| markdown | 编辑markdown文档 |
| desc     | 添加描述 |
